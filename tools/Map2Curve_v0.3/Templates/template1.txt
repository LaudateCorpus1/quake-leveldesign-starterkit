//=============== MAP2CURVE SETTINGS FILE ======================================++
//										||
// --- Settings File ---------------------------------------------------------- ||
// 1. The Settings-File needs to have the same name as the *.map-File		||
//										||
// --- Commands --------------------------------------------------------------- ||
// 1. The only command that is crucial to generate an ARC, is "rad".		||
//    Every rad command, stands for one arc object.				||
// 2. You can generate multiple arcs from the same map-file. To do this,	||
//    just add more than one rad command.					||
// 3. The Order of the commands specifies to which arc object they belong.	||
// 4. If there are e.g. 5 "rad"-commands, you only need to specify one optional	||
//    command (offset, res, type, arc, obj).					||
//    The following arc objects then share the settings of the first arc,	||
//    use default values or automatically generate them from previous values.	||
// 5. If there are e.g. more "res"-commands than "rad"-commands, the redundant	||
//    res-commands will be ignored.						||
//										||
//    Commands can be commented out like this: "// rad 1024"			||
//------------------------------------------------------------------------------++

// Radius
rad 0		// Radius of first arc: "0" uses original radius (y-axis-coordinates) from map-file.
rad 768		// Custom radius for 2nd...
rad 1536	// ...and third arc.

// Offset
// Additional offset on top of the radius (useful if an arcs uses "rad 0" and therefor original radius is unknown).
offset -32
offset 16
offset 64

// Resolution for arc objects (overall sides the arc shall have)
// PI and GRID arcs need different amount of sides!
// "0" uses resolution of last arc as a base and maintains its "level of detail" for the new arc radius.
res 16
res 24
res 0

// Method for generation of construction-circles
// "0" use PI to generate the construction circles...
// "1" uses the GRID method.
type 0

// Arc Export Range
// How many degrees of the full arc will be exported to the Map-File
arc 90
arc 90
arc 45

// Horizontal Texture Shift Mode
// 0 = Null-Shift, 1 = Per Section, 2 = Per Brush, 3 = Per Brush Texture,
// 4 = Align Left, (5 = Align Right - not implemented yet)
shift 3

// OBJ-File Export
// This will export a single arc object into a seperate *.obj-File.
// "OBJ" is a model exchange format readable by most 3D software.
//obj 1

// Triangulation
// Needed for Ramp-Generation and only works for Brushes with 6 faces at the moment (Cuboids).
// Top and Bottom Faces can be sloped though.
// 0 = Off, 1 = On
tri 0
tri 1
tri 0

// Custom Height per section
// This height is being added to each section which will lead to a stair. Can be combined with "ramp".
// 0 = Off
height 0
height 32
height 0

// Ramp Generation
// Will create a sloped ramp from previously chosen "height" setting.
// Can have 2 different slope methods: linear and smooth.
// Works with 6 sided Brushes (Cuboids) only at the moment and needs to have triangulation (tri) activated.
// Some Texture Alignments will be faulty. It is advised to "ramp" non-sloped Brushes only and manipulate
// vertices after generation to match the desired shape.
ramp 0
ramp 1
ramp 0


// Round coordinates to integer numbers
// Will round floating point coordinates like 5.3123895483 to nearest integer number -> 5
// 0 = Off, 1 = On
round 0
